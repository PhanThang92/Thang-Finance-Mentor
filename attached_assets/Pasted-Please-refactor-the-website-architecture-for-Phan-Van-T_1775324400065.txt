Please refactor the website architecture for “Phan Van Thang SWC” into a cleaner hybrid content system.

IMPORTANT:
- Write implementation logic and code changes in English.
- Keep all visible UI text in Vietnamese.
- Do not break the premium frontend design that has already been built.
- The goal is not only to make the site work now, but to make it scalable and maintainable later.

GOAL:
Refactor the current content/data structure so the site uses a hybrid model:

1. Static structured content config for stable brand pages and reusable site copy
2. Data collections for scalable content sections like articles, videos, topics, series, and news
3. Clean UI mapping layer so pages render from organized data sources instead of messy hardcoded content

This should make the website easier to maintain, easier to scale, and ready for future database or CMS integration.

================================
1. ARCHITECTURE STRATEGY
================================

Please reorganize the content system into 3 layers:

LAYER A — STATIC / STRUCTURED CONTENT CONFIG
Use this for content that is mostly stable and brand-controlled.

Examples:
- About page content
- homepage CTA text
- footer brand description
- site-wide contact blocks
- navigation labels and dropdown structure
- reusable section labels / microcopy if appropriate

LAYER B — CONTENT COLLECTIONS
Use this for repeatable and scalable content.

Examples:
- articles
- videos
- topics
- series
- news
- featured items

LAYER C — UI MAPPING LAYER
All pages and sections should render by reading from organized data/config rather than hardcoded content directly inside components.

Examples:
- homepage sections map from data
- article listing maps from article collection
- video library maps from video collection
- knowledge hub maps from topics / series / featured content

================================
2. WHAT SHOULD NOT BE FULL DATABASE-DRIVEN YET
================================

Do NOT over-engineer everything into a database immediately.

For now, keep the About page and stable brand sections as structured content config files, not full database-driven entities.

This includes:
- `/gioi-thieu`

These should be stored in well-structured config/data files so they are easy to edit without touching layout code.
