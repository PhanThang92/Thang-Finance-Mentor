Please build a shared Media Library inside the existing admin CMS for “Phan Van Thang SWC”.

IMPORTANT:
- Write implementation and code in English.
- Keep all visible UI text in Vietnamese.
- Build on top of the current admin CMS, Supabase/Postgres, Supabase Storage, and existing image processing pipeline.
- Do NOT redesign the whole admin system.
- Focus specifically on creating a reusable media library / asset manager for uploaded images.

GOAL:
Create a centralized Media Library in the admin CMS so editors can:
- view uploaded images in one place
- search and filter assets
- reuse existing assets across Articles, Videos, Topics, Series, and future content types
- inspect metadata
- copy URLs
- select assets from a media picker inside forms
- replace or delete assets safely
- keep media storage better organized over time

This should turn media handling into a reusable system, not just a per-form upload field.

================================

Preferred route:
- `/admin/media`

This module should support:
- viewing all uploaded media assets
- filtering media by type and usage
- searching by filename, alt text, title, or related content
- selecting an asset from a modal/picker when editing content
- previewing asset details
- deleting or replacing assets where safe
- reusing assets across multiple content entries

Primary target:
- images only for now

Future-ready target:
- could later support OG images, topic covers, series covers, banners, etc.
