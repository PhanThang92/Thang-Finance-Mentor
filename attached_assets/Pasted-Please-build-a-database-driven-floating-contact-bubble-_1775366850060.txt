Please build a database-driven floating contact bubble widget for the public website of “Phan Van Thang SWC”.

IMPORTANT:
- Write implementation and code in English.
- Keep all visible UI text in Vietnamese.
- Build this feature in a clean, scalable, maintainable way.
- Do NOT hardcode phone/Zalo/Telegram values directly in frontend components.

GOAL:
Create a floating contact widget that appears in the bottom-right corner of the public website.

The widget should:
1. show a primary floating bubble button
2. expand to reveal 3 direct contact options:
   - gọi số điện thoại
   - chat Zalo
   - Telegram
3. read all channel data from the database
4. be manageable from the admin CMS
5. support desktop and mobile behavior cleanly
6. support tracking/analytics for clicks
7. feel premium and non-intrusive

================================

Do NOT hardcode contact values in the frontend.

B. contact channel items

A. `site_contact_widget_settings`
Fields:

- is_enabled (boolean)
- widget_title (nullable)
- widget_subtitle (nullable)
- position (text, default "bottom-right")
- expand_direction (text, default "up")
- show_labels (boolean)
- show_tooltips (boolean)
- desktop_offset_x (integer or text)
- desktop_offset_y (integer or text)
- mobile_offset_x (integer or text)
- mobile_offset_y (integer or text)
- default_open (boolean, default false)
- theme_variant (nullable)
- show_on_desktop (boolean)
- show_on_mobile (boolean)
- created_at
- updated_at

B. `contact_channels`
Fields:
- id
- channel_type (text: phone / zalo / telegram)
- label (text)
- value (text)
- icon_key (text, nullable)
- tooltip_text (text, nullable)
- is_enabled (boolean)
- display_order (integer)
- open_mode (text: tel / same_tab / new_tab)
- show_on_desktop (boolean)
- show_on_mobile (boolean)
- created_at
- updated_at

Requirements:
- the widget must render from DB values
- only enabled channels should show
- channels should be sorted by display_order
- if no settings or no enabled channels exist, fail gracefully and hide widget
